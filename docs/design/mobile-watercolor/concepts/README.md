## 已批准视觉方向

- 仅使用固定浅色，页面纸张底色为 `#faf8f5`。
- 水彩只服务品牌氛围；工具表面、反馈状态和媒体画布保持实色、中性、可验证。
- 图鉴首页的模板提炼入口是唯一强水彩焦点；长列表项不重复加载纹理。
- 图鉴条目顶部允许水彩品牌表面，其后的模型、输入、规格和提交栏保持中性。
- 图片及图片缺失占位固定落在 `#f1f2f3` 中性 media matte，不使用色罩、滤镜、混合模式或纸纹。
- 主操作为不透明蓝灰 `#4a6fa5`；错误、警告和成功继续使用独立语义色。
- 正文、按钮、导航、状态和数据使用平台系统 sans，不引入中文手写字体或全局斜体。
- 手机横向 gutter 为 20px；平板/Web 内容最大宽度为 720px并居中。
- 圆角：品牌面板 22px、标准卡片 18px、工具卡 16px、字段和按钮 14px、媒体框 12px、共享徽章 pill。
- 默认卡片不使用明显阴影；原生 Stack 与 Tab 只接入配色，不放水彩资产。

## 已批准的生产纠正规则

生成图用于冻结视觉关系，不直接作为像素级生产 UI。生产实现按以下规则纠正生成模型的偏差：

1. `catalog-empty-mobile.png` 的空状态插画面积偏大，生产空状态应降低其尺寸和存在感，确保模板提炼入口仍是唯一强焦点。
2. 概念图中部分图片详情按钮的可见尺寸约为 38px；生产实现的真实命中区必须至少为 44×44pt。
3. `catalog-tablet.png` 的媒体框视觉比例接近但不完全等于 16:9；生产 `MediaFrame` 必须使用精确 16:9。
4. `catalog-loading-mobile.png` 的反馈面板偏高；生产加载反馈采用内容自适应的紧凑面板，不冻结该高度。
5. `template-refinement-review-mobile.png` 的禁用主按钮仅表达状态关系；生产 disabled variant 必须单独验证文字对比度。
6. 概念图内的示例图片、日期和记录内容只是构图数据，不作为生产资产或新增业务 fixture。
7. 字形、字距和原生导航细节以各平台实际系统组件为准，不复制生成图中的伪原生像素。

## 文案与信息架构边界

- 生产实现以当前代码中的真实文案、顺序、路由和状态为准。
- 概念稿没有批准新增营销文案、统计卡、搜索、侧栏或新工作流。
- `catalog-tablet.png` 的待审阅入口使用真实说明“有一份提炼方案等待确认写入。”。
- 加载、错误和进行中状态不新增重试、取消、百分比或预计时间。
- 审阅工具页保持“写入信息 → 方案摘要 → 提炼补充 → 输入声明 → 完整正文 → 批准项 → 操作”的现有顺序。
