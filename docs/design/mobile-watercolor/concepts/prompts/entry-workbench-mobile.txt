创建一张 390×844 像素、纵向、完整未裁切的移动应用高保真 UI 概念稿。它是 Expo React Native 应用 Imagemon 的“图鉴条目”工作台页面，不是网页、海报、营销页或手机外壳。必须像真实原生应用截图，所有中文和英文必须清晰准确。

页面用“水彩品牌层 + 中性工具层”的明确分界。固定浅色：纸张背景 #faf8f5，主表面 #fffdfb，抬升工具表面与字段 #ffffff，正文 #3a3430，次文字 #6f6964，主操作 #4a6fa5，细边框 rgba(74,111,165,0.18)，中性媒体画布 #f1f2f3。系统 sans 字体，20px 横向 gutter，品牌卡圆角 22px，工具卡圆角 16px，表单控件圆角 14px，极弱或无阴影。禁止手写字体、斜体、玻璃拟态、纸纹覆盖、彩色表单、渐变按钮和过重阴影。

顶部是原生 Stack 导航栏，左侧返回箭头，居中标题“图鉴条目”。导航栏保留平台质感，不放水彩资产。

滚动内容顶部是唯一品牌表面：近白实色大卡，外缘背后有低透明度蓝灰、青绿和少量蜜桃的柔和水彩 wash，但装饰绝不穿过文字。卡片内按现有顺序展示：主标题“light-infographic”；徽章“内置”；同一信息组中徽章“生成”和状态文字“可执行”；说明“将一段文字转换为浅色、清爽、结构清晰的解释性信息图”。品牌区温暖、有呼吸感，但仍清晰专业，不添加副标题或营销文案。

品牌区下方立即进入边界清楚的中性工具层，所有表面纯白或近白，无水彩、无纹理。先显示工具卡“图片模型”，左侧小图片图标，内部是一行已配置模型摘要“gpt-image-2”，右侧 chevron。然后显示工具卡“任务输入”：第一项标签“内容”，右侧“必需”，说明“需要转换为单张配图的一段文字”，下面是 132px 高的白色输入区域，placeholder 仍为同一句说明，底部显示“0 字”和 expand 图标；第二项标签“辅助标题”，右侧“可选”，露出其输入区域顶部。再往下只露出“图片规格”区块标题和三个等宽选择块的顶部，保持页面真实可滚动。

底部固定一条中性实色提交栏，顶部细分隔线，内有一个不透明蓝灰主按钮，白字“生成图片”，左侧 sparkles 图标；按钮高度至少 48px，不透明、不渐变。工具表单、状态、按钮必须清楚可验证，水彩严格停留在顶部品牌区。不要添加生成统计、教程、提示卡或新工作流。
