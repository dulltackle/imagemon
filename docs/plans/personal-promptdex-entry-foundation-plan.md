# 个人图鉴条目基础能力实现计划

本文描述手机端为模板提炼写入先行补齐个人图鉴条目与合并图鉴基础能力的实现计划。目标是让设备使用者经模板提炼确认后可以写入个人图鉴条目，并让该条目像内置图鉴条目一样出现在图鉴列表、详情页和图片任务执行路径中。

## 已确认范围

- 本轮把个人图鉴条目持久化与合并图鉴能力纳入模板提炼前置范围，并且先做它。
- 个人图鉴条目以结构化字段作为真相源，不把整份 Promptdex Markdown 作为不透明文本保存。
- 本轮表结构只包含 Promptdex Markdown 契约字段：`name`、`description`、可选 `version`、`inputs` 和模板正文。
- 标题、用途说明、图鉴分类、搜索标签和封面引用等条目展示信息属于既有产品模型，但本轮不实现，也不提前占位。
- 本轮个人图鉴条目的写入入口只服务模板提炼确认写入，不做手动新增、手动编辑执行语义或从 Markdown 导入 UI。
- 仓储层提供硬删除能力，合并图鉴只读取当前存在的个人图鉴条目；本轮不开放删除 UI。
- 合并图鉴列表按来源分组，个人图鉴条目在前，内置图鉴条目在后；组内按名称升序。
- 列表项显示来源徽标：`个人` 或 `内置`。
- 如果个人图鉴条目名称与内置图鉴条目名称冲突，合并图鉴保留个人条目并抑制同名内置条目，符合 ADR 0186。
- 图鉴详情页继续使用 `/promptdex/[name]` 路由，不把来源类型放入 URL；详情页按名称从合并图鉴查找当前可用条目。
- 个人图鉴条目被删除后，再打开对应 URL 显示图鉴条目不存在；任务历史仍由任务快照解释，不追随当前图鉴。

## 明确不做

- 不新增个人图鉴管理页。
- 不新增手动创建个人图鉴条目入口。
- 不新增从剪贴板或文件导入图鉴条目。
- 不允许设备使用者直接编辑图鉴条目的名称、输入声明或模板正文。
- 不实现条目展示信息编辑。
- 不实现封面示例图选择、引用或迁移。
- 不实现图鉴分类筛选、搜索标签搜索或按展示标题排序。
- 不开放删除 UI、删除确认弹窗或删除后导航规则。
- 不实现 ZIP 备份和恢复中的个人图鉴条目迁移。

## 依赖决策

- `CONTEXT.md`：提示词图鉴、合并图鉴、图鉴条目、内置图鉴条目、个人图鉴条目、Promptdex Markdown、模板提炼。
- ADR 0010：图鉴条目名称在合并图鉴中全局唯一。
- ADR 0020：首版个人图鉴条目不提供停用或隐藏状态。
- ADR 0022：模板提炼确认后写入新的个人图鉴条目。
- ADR 0053：个人图鉴条目展示信息后续可编辑，但执行语义不可直接编辑。
- ADR 0185：模板提炼写入时的名称冲突由写入前改名解决。
- ADR 0186：升级引入的同名内置图鉴条目在合并图鉴中被抑制。
- ADR 0206：个人图鉴条目初始存储只包含 Promptdex 契约字段。

## 数据模型

初始 SQLite 表建议为 `personal_promptdex_entries`：

- `name TEXT PRIMARY KEY`
- `description TEXT NOT NULL`
- `version_json TEXT`
- `inputs_json TEXT NOT NULL`
- `body TEXT NOT NULL`
- `created_at TEXT NOT NULL`
- `updated_at TEXT NOT NULL`

规则：

- `name` 必须满足 Promptdex 模板名称规则，并与合并图鉴中的当前可用条目唯一。
- `inputs_json` 保存 Promptdex 输入声明结构，读取时必须重新进入 core 校验。
- `version_json` 只在原模板显式包含 `version` 时保存；没有版本时为空。
- 写入前用 `validatePromptdexTemplate` 或等价 core 解析路径校验 Promptdex 契约。
- 导出或复制 Promptdex Markdown 时从结构化字段恢复为 `PromptdexTemplate` 后序列化。

## 合并图鉴行为

合并图鉴仓储读取内置图鉴条目和个人图鉴条目，并产出统一条目列表与按名称查找能力。

列表排序：

1. 个人图鉴条目，按 `name` 升序。
2. 内置图鉴条目，按 `name` 升序。

冲突处理：

- 正常写入路径不允许创建与任何当前合并图鉴条目同名的个人图鉴条目。
- 如果既有个人条目与应用包内置条目同名，合并图鉴显示个人条目，不显示同名内置条目。
- 被抑制内置条目不把正文、输入声明或展示信息泄漏到个人条目上。

详情页：

- `/promptdex/[name]` 继续只携带名称。
- 查找命中后显示来源徽标。
- 执行图片任务时，任务快照中的来源类型写入 `personal` 或 `built-in`。

## 可执行子任务与提交边界

### 1. 记录领域语言、ADR 与实现计划

Commit 标注：`docs: 记录个人图鉴条目基础范围`

改动范围：

- 在 `CONTEXT.md` 增加 `合并图鉴`。
- 新增 ADR 0206，记录初始存储只包含 Promptdex 契约字段。
- 新增本文档，冻结范围、排序、路由和提交拆分。

验收：

- 文档明确区分本轮基础能力和后续个人图鉴管理能力。
- 文档明确本轮不改变条目展示信息的既有产品决策。

### 2. 增加个人图鉴条目 SQLite schema

Commit 标注：`feat: 增加个人图鉴条目存储`

改动范围：

- 将应用 schema 版本递增。
- 新增 `personal_promptdex_entries` 表和从上一版本迁移的 migration。
- 补齐 storage 初始化测试，覆盖新表创建和迁移执行。

验收：

- 新安装数据库直接创建个人图鉴条目表。
- 旧版本数据库升级后拥有个人图鉴条目表。
- 不修改既有测试断言、Mock、Fixture 来绕过失败。

### 3. 实现个人图鉴条目仓储

Commit 标注：`feat: 实现个人图鉴条目仓储`

改动范围：

- 新增 `apps/mobile/src/promptdex` 下的个人图鉴条目 repository。
- 提供 `list`、`get`、`saveFromTemplate` 和 `delete`。
- 内存 store 和 SQLite store 保持与现有 repository 风格一致。
- 写入时校验 Promptdex 契约字段，不写入无效模板。
- 删除为硬删除。

验收：

- 单元测试覆盖写入、读取、列表排序、重复名称拒绝、无效模板拒绝和删除。
- 仓储不保存外部完整提示词、提炼模型原始响应或条目展示信息。

### 4. 实现合并图鉴服务

Commit 标注：`feat: 合并个人与内置图鉴条目`

改动范围：

- 把当前内置图鉴加载逻辑扩展为合并图鉴服务或仓储。
- 合并内置条目和个人条目，生成统一列表项。
- 列表项包含来源类型、任务类型、输入摘要、执行状态和来源徽标所需字段。
- 按来源分组排序，并抑制与个人条目同名的内置条目。
- 提供按名称查找当前可用图鉴条目的能力。

验收：

- 个人条目排在内置条目前，组内按名称升序。
- 同名冲突时只返回个人条目。
- 按名称查找不需要来源类型即可唯一命中。

### 5. 接入 AppRuntime

Commit 标注：`feat: 注入个人图鉴条目运行时仓储`

改动范围：

- 在 `AppRuntimeProvider` 初始化个人图鉴条目仓储和合并图鉴服务。
- Web 非安全上下文内存运行时也提供等价能力。
- 保持现有模型配置、图片任务和文件存储初始化行为不变。

验收：

- 应用运行时 ready 状态包含个人图鉴条目相关能力。
- 现有页面不因新增运行时字段破坏类型检查。

### 6. 更新图鉴列表和详情页

Commit 标注：`feat: 图鉴页面读取合并图鉴`

改动范围：

- 图鉴列表页从合并图鉴读取条目，不再只读内置图鉴。
- 列表项显示 `个人` / `内置` 来源徽标。
- 详情页继续使用 `/promptdex/[name]`，按名称从合并图鉴查找条目。
- 详情页执行图片任务时传入正确 `sourceType`。
- Promptdex Markdown 复制继续从当前模板对象序列化。

验收：

- 内置条目现有执行路径保持可用。
- 个人条目可出现在列表、打开详情并执行图片任务。
- 删除后的个人条目 URL 显示不存在。
- 任务快照记录正确来源类型。

### 7. 为模板提炼写入预留接口验收

Commit 标注：`test: 验证个人图鉴条目可服务提炼写入`

改动范围：

- 增加一个不依赖 UI 的测试，模拟模板提炼确认后的 `PromptdexTemplate` 写入个人图鉴条目仓储。
- 验证写入后合并图鉴可按名称找到该条目，并可用于 Promptdex 图片任务快照。

验收：

- 提炼写入所需最小仓储接口已具备。
- 不新增手动创建或导入 UI。

## 推荐实施顺序

1. 先提交文档、术语和 ADR。
2. 再做 SQLite schema 和 storage 测试。
3. 然后做个人图鉴条目 repository。
4. 接着做合并图鉴服务。
5. 把仓储和合并服务注入 AppRuntime。
6. 最后更新图鉴列表、详情页和任务来源快照。

## 风险与注意事项

- 不要把本轮“暂不存展示信息”理解成废弃展示信息产品模型；后续仍要通过 migration 补齐。
- 不要允许个人条目遮蔽内置条目写入；只有升级造成的既有冲突才按 ADR 0186 抑制内置条目。
- 不要把路由改成来源加名称；名称全局唯一已经是合并图鉴边界。
- 不要在个人条目仓储中保存外部完整提示词或提炼过程。
- 不要让导入、导出、删除 UI 混入本轮基础提交。
