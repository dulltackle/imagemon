# 移动端 UI/UX 全面盘点（2026-07-13）

## 方法与范围

- **基线**：`main` @ v0.10.0，Android 模拟器 `imagemon-avd`（Pixel 类手机、竖屏、浅色模式）。
- **证据**：实机截图 + 代码交叉核对。截图使用现有 `scripts/mobile-android-screenshots.mjs`（SCREENSHOT_MODE 种子数据），存于本地 `apps/mobile/.expo/screenshots/android/ux-audit-baseline/`（原始种子）与 `ux-audit-with-image/`（临时补充真实图片后重截，补丁已还原、未提交），**截图不入库**。
- **评判标准**（每条问题标注命中项）：
  - **可用性**：通用可用性启发式（状态可见、一致性、容错、效率等）；
  - **术语**：界面文案对照 `CONTEXT.md` 术语表与 Avoid 列表；
  - **平台**：Android/Material 惯例，仅作参考不作硬标准；
  - **ADR**：与已接受 ADR 的偏差（硬性依据）。
- **严重度**：P0 = 崩溃或核心闭环断裂；P1 = 明显伤害核心体验或领域承诺缺失；P2 = 打磨与一致性。
- **修复成本**：小（≤半天）/ 中（1–3 天）/ 大（需单独设计）。

深色模式、大屏适配未实机走查（按约定只截常规态），相关问题来自代码走查，已单独标注。

## 一、条目详情 / 任务表单（生成任务闭环核心）

`src/promptdex/PromptdexEntryDetailScreen.tsx`

### 1.1 【P0·可用性】首次生成成功后界面崩溃（条件 Hooks）
`EntryImagesSection`（约 1075 行）在 `if (images.length === 0) return null;` **之后**才调用 `useCSSVariable` 三个 hooks。对一个从未生成过图片的条目首次生成成功时，`mergeEntryImages` 使 `images` 从 0 变 1，同一组件实例的 hooks 数量从 0 变 3，违反 React hooks 规则直接抛错——恰好发生在用户第一次生成成功的时刻。
**建议**：把 hooks 调用移到 early return 之前。**成本：小**。

### 1.2 【P0·ADR 0037/0038 偏差】应用默认规格链路断裂
- 表单尺寸初始值硬编码 `useState<ImageTaskSize>("1024x1024")`（93 行），未读取应用默认规格；
- 设置页没有「应用默认规格」编辑入口（ADR 0038 明确"使用者可以在设置中修改应用默认规格"）；
- 质量 / 格式 / 数量在表单中完全不可调（ADR 0038："也可以在执行前只修改本次任务规格"，规格含四个维度）。
**建议**：设置页增加应用默认规格编辑；表单从设置预填，并按产品判断暴露其余维度。**成本：中**。

### 1.3 【P1·可用性】任务表单被压到两屏之外，首屏无行动点
首屏顺序：条目信息卡 → 生成图片区（大图+缩略列表，占满一屏多）→ Promptdex Markdown → 图片模型 → 任务输入 → 尺寸 → 提交按钮。到这一屏的核心意图是发起任务，但首屏看不到任何输入或 CTA（见 `ux-audit-with-image/promptdex-built-in-detail.png`）。
**建议**：任务表单上移（生成图片区收为横向缩略条或折叠），或提交按钮吸底常驻；Markdown 移至更深层级。**成本：中**。

### 1.4 【P1·可用性-效率】任务输入编辑要点两次
点输入卡片打开 Modal 后是只读预览态，还要再点右上「编辑」才出现输入框和键盘；键盘收起又退回预览态。移动端填写路径多出一步无收益。
**建议**：Modal 打开即进入编辑态并 autoFocus。**成本：小**。

### 1.5 【P1·可用性-状态可见】提交按钮禁用不说明原因
必填未填、模型未配置、锁被占用时按钮同样变灰，仅模型缺失场景有橙色提示。
**建议**：按钮上方给出当前不可提交的具体原因（如"必填输入 content 未填写"）。**成本：小**。

### 1.6 【P2·一致性】尺寸选中态硬编码 `bg-blue-50`（895 行）
全应用唯一一处直接用 Tailwind 调色板而非 `sf-*` 语义色；深色模式下仍是亮色底（代码走查判断，未实机验证深色）。**成本：小**。

### 1.7 【P2·冗余】代表图连续展示两遍
生成图片区大图正下方第一条缩略行就是同一张「代表图」。**建议**：缩略列表跳过代表图或合并展示。**成本：小**。

### 1.8 【P2·可用性/一致性】大图不可点，只有右上角 38px 小按钮
详情页大图本体无 onPress；而首页已生成卡片上点大图是进条目、点小按钮才是看图——同一视觉元素两种行为约定。**建议**：统一为"点图看图"，卡片进入条目用卡片其余区域。**成本：小**。

### 1.9 【P2·冗余】输入说明文案出现两遍
输入卡片上方显示 `input.description`，未填时卡片内 placeholder 又是同一段文案。**成本：小**。

## 二、全局 / 跨界面

### 2.1 【P1·ADR 0173/0174 缺失】全局模型调用状态不存在
`ModelCallLockProvider` 只有锁语义，没有任何全局可见的状态 UI。业务调用进行中导航离开后（ADR 0156 调用继续），用户没有任何指示，也没有"返回当前调用页面的入口"（ADR 0174、术语「全局模型调用状态」）。等待期间的体验完全依赖用户停在原页面。
**建议**：实现全局状态条/胶囊（显示调用类型 + 点击回到调用页）。**成本：大（需设计）**。

### 2.2 【P1·ADR 0159–0172 缺失】业务调用提示（角标）未实现
成功结果、失败与待处理项没有任何入口级/列表项级可发现性提示；ADR 0159–0172 定义了完整的角标行为矩阵。
**建议**：与 2.1 一起规划实现。**成本：大（需设计）**。

### 2.3 【P1·可用性/术语·实机确认】日期时间格式为英文且前后不一致
实机显示 `01/01, 7:04 PM`（首页、历史列表，无年份）与 `01/01/2026, 7:04 PM`(详情页，有年份)。代码虽传 `toLocaleString("zh-CN")`，Hermes 实际输出英文格式；且各文件各自复制 `formatDateTime`，年份取舍不一。中文 UI（ADR 0064）中非常突兀。
**建议**：抽一个共享的中文日期格式化 helper（不依赖 ICU，手写拼接如 `2026-01-01 19:04`），统一含年策略。**成本：小**。

### 2.4 【P2·一致性】双强调色
Tab 选中色硬编码 `#0F766E`（teal），应用内强调色为 `--sf-blue`（iOS 蓝）。同屏出现两套主色（见 `catalog.png`：teal 的 Tab + 蓝色的提炼卡边框）。**建议**：二选一，写入语义变量。**成本：小**。

### 2.5 【P2·一致性】组件样式细碎不一致
- 徽章圆角：历史状态徽章 `rounded-full`，首页来源/类型徽章 `rounded-lg`；
- 历史列表卡有 `shadow-sm`，其余卡片无阴影；
- 反馈样式：条目详情用带图标的边框卡片，模型配置编辑器用裸彩色文本；
- 字重：首次设置/编辑器用 `font-semibold/bold`，其余界面标题为 `font-extrabold`。
**建议**：沉淀 Badge/FeedbackBox/SectionTitle 等共享组件。**成本：中（一次性收敛）**。

### 2.6 【P2·平台】几乎所有 Text 都 `selectable`
含按钮标签、徽章、日期等非正文文本。Android 上长按选中与 Pressable 手势易冲突，也不符平台惯例。**建议**：仅正文/提示词类文本保留。**成本：小**。

### 2.7 【P2·ADR 0062 缺口·代码走查】无最大内容宽度
全部界面无 `max-w-*`，大屏内容会无限拉伸，ADR 0062 要求基础大屏适配。**成本：小–中**。

## 三、任务历史

`app/(tabs)/(history)/history.tsx`、`app/history/[id].tsx`

### 3.1 【P1·ADR 0026/0187 缺失】失败/状态未知任务没有「重新填写」入口
历史详情对失败任务只展示失败摘要，没有进入当前可用图鉴条目新表单并预填历史输入的动作（领域术语「重新填写」）。闭环容错断点：失败后用户只能自己回图鉴找条目重抄输入。
**建议**：失败/未知详情页增加「重新填写」按钮 → 条目表单带预填。**成本：中**。

### 3.2 【P1·ADR 0041 缺失】完整提示词无复制按钮
历史详情完整提示词区只能靠长按选择文本；领域定义了显式动作「完整提示词复制」。**建议**：区块右上加复制按钮（复用 Markdown 复制的控件模式）。**成本：小**。

### 3.3 【P2·术语】「失败摘要」应为「错误摘要」；原始枚举直接展示
- 区块标题用「失败摘要」，CONTEXT.md canonical 术语是「错误摘要」；
- `原因 network_error`、`质量 auto` 等英文原文直接面向设备使用者。
**建议**：术语对齐 + reason/quality 中文标签映射。**成本：小**。

### 3.4 【P2·可用性】失败项缩略图暗示有图
失败/未知任务列表缩略位用蓝色图片图标占位，视觉上像"有图未加载"。**建议**：按状态用差异化占位（如警示图标）。**成本：小**。

## 四、图片详情

`app/images/[id].tsx`

### 4.1 【P1·可用性】图片不可缩放、无全屏查看
`maxHeight: 520` 内联展示，无 pinch-zoom/双击放大。图片生成应用里无法查看细节。**建议**：点图进全屏可缩放查看器。**成本：中**。

### 4.2 【P2·领域覆盖】导出只有「保存到系统相册」
CONTEXT.md「图片结果导出」定义含分享、保存相册、导出到文件系统三种；现仅一种。**成本：中（roadmap）**。

### 4.3 【P2·术语】「基础规格」非领域术语
就近术语是「图片规格」。**成本：小**。

## 五、资产管理缺口（跨界面，多为 roadmap）

### 5.1 【P1·领域缺口】删除操作整体缺失
个人图鉴条目、任务历史、图片结果均无删除入口，而领域模型明确三者"长期保留至设备使用者主动删除"（并有 ADR 0014/0137 等配套决策）。目前只有模型配置可删。**成本：中**。

### 5.2 【P2·ADR 缺口（已知 roadmap，列出备忘）】
- 个人图鉴条目：编辑展示信息（ADR 0053）、单条导出文件（ADR 0044）、Markdown 导入（ADR 0005/0061）无 UI；
- 搜索/筛选（ADR 0049；ADR 0209 已声明合并首页首个闭环不含搜索，属已决策延后）；
- 设置页：ZIP 备份/恢复入口（ADR 0008/0061）、应用默认规格（见 1.2）缺失；
- 编辑任务的蒙版类条目仅提示「蒙版编辑后续支持」（一致，无问题）。

## 六、首次设置

`src/first-run/index.tsx`

### 6.1 【P1·可用性】零上下文直入表单
首屏直接是 Base URL/模型名/API Key，没有一句话说明应用是什么、为什么需要模型配置、跳过会怎样；「完成/跳过」在折叠线下（见 `first-run.png`）。ADR 0065 只排除营销式 onboarding，不排除一句说明。
**建议**：顶部加 1–2 句引导文案；说明跳过后可随时在设置中配置。**成本：小**。

### 6.2 【P2·可用性】「使用相同连接信息」开关语义模糊
只在切换瞬间复制图片模型的 Base URL/API Key，之后不同步，文本区仍可编辑。**建议**：开启时锁定文本模型对应字段并持续同步，或改为一次性「复制图片模型连接信息」按钮。**成本：小**。

## 七、模型配置

`app/model-configurations/*`、`src/model-configurations/ModelConfigurationEditor.tsx`

### 7.1 【P1·术语+层级】「保存草稿」按钮
- 领域中没有"模型配置草稿"概念（草稿是「提炼草稿」专属，撞词）；ADR 0033 的说法是"未就绪也可保存"；
- 主按钮（蓝）给了保存草稿，「保存并测试」反而是次级——与首次设置（保存并测试是唯一主操作）主次相反，而后者才是通往「就绪模型配置」的正路。
**建议**：文案改「保存」，主次对调。**成本：小**。

### 7.2 【P2·可用性】新建时展示「未保存凭据 + 清除凭据」
新建场景没有可清除的凭据，这一行只制造困惑。**建议**：仅编辑已有配置且 `hasCredential` 时显示。**成本：小**。

### 7.3 【P2·ADR 0146/0175 缺口】测试连接无取消
ADR 暗示测试可取消（取消保留测试状态、全局状态入口保留取消能力），编辑器测试中只能干等 30 秒超时。**成本：中**。

### 7.4 【P2·视觉层级】列表页两个「新建」按钮占顶部主视觉
新建是低频操作却是页面最重元素，且一蓝一灰主次无依据。**建议**：移到列表尾部或导航栏 + 统一样式。**成本：小**。

## 八、模板提炼

`src/promptdex/TemplateRefinementScreen.tsx`

### 8.1 【P2·术语/ADR 0064】写入信息字段标签是英文小写 `name`、`description`
同屏其他标签全中文。**建议**：「名称」「说明」，必要时括注契约字段名。**成本：小**。

### 8.2 【P2·可用性】外部完整提示词无「粘贴」快捷按钮
该输入几乎总是从剪贴板粘贴而来，placeholder 也写着"粘贴完整提示词"。**建议**：输入框旁加一键粘贴。**成本：小**。

### 8.3 【P2·文案与行为不符】「重新生成」实际是返回编辑
按钮点击只切回输入编辑态，并不直接重新生成。**建议**：改文案「返回修改输入」或真的直接重发。**成本：小**。

## 建议优先级（第一权重：生成任务闭环主路径影响）

**第一批：闭环上的断点与硬伤（建议尽快）**
1. **1.1** 首次生成成功崩溃（P0，成本小）
2. **1.2** 应用默认规格链路（P0，成本中；含设置页入口）
3. **3.1** 失败/未知任务「重新填写」（P1，成本中）
4. **1.4** 输入编辑两步变一步（P1，成本小）
5. **1.5** 提交禁用原因提示（P1，成本小）
6. **1.3** 条目详情信息架构调整（P1，成本中）

**第二批：全局可见的体验与承诺（建议随后）**
7. **2.3** 中文日期格式统一（P1，成本小，全局可见）
8. **3.2** 完整提示词复制（P1，成本小）
9. **7.1** 「保存草稿」术语与按钮主次（P1，成本小）
10. **6.1** 首次设置引导文案（P1，成本小）
11. **4.1** 图片全屏缩放查看（P1，成本中）
12. **2.1 + 2.2** 全局模型调用状态 + 业务调用提示（P1，成本大，建议合并为一个独立设计任务）
13. **5.1** 三类资产删除入口（P1，成本中，或并入 roadmap）

**第三批：一致性打磨簇（可打包为一个「视觉收敛」任务）**
14. **2.4** 双强调色 → **2.5** 组件收敛 → **1.6/1.7/1.8/1.9** 条目详情细节 → **3.3/3.4/4.3/8.1/8.3** 术语与文案 → **2.6** selectable 收敛 → **7.2/7.4** 模型配置细节 → **6.2** 开关语义 → **8.2** 粘贴按钮。

**Roadmap 备忘（非缺陷，按产品节奏排期）**：5.2（个人条目管理/导入导出、搜索、ZIP 备份入口）、4.2（导出方式补全）、7.3（测试取消）、2.7（大屏 max-width）。
