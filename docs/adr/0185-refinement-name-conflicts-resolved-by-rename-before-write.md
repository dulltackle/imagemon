# 提炼写入时的名称冲突由写入前改名解决

模板提炼确认写入个人图鉴条目时，如果拟用模板名与当前合并图鉴中的内置图鉴条目或个人图鉴条目冲突，应用不覆盖、不遮蔽、不静默改名，也不直接丢弃提炼成果，而是让使用者在写入前修改拟用模板名后再写入。这把 0010 中"提炼时名称冲突拒绝写入"细化为与导入（0005、0035）一致的"写入前改名"路径：拒绝的是覆盖或遮蔽，而不是拒绝整个提炼。

按提炼方案格式，修改模板名会使原确认失效，需要展示更新后的方案并重新等待一次本地确认；重新确认是本地动作，不是 0184 意义上的"重新生成方案"，不发起模型调用。改名后写入前做最终唯一性校验，新名称须通过图鉴条目契约校验，且不得与当前合并图鉴再次冲突。写入后的个人图鉴条目仍保持执行语义不可变（0020 / 0053）。
