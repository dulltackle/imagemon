# 模型配置测试失败摘要只在页面会话内显示

手机端不再持久保存模型配置测试失败摘要。模型配置测试连接失败后，当前首次设置或模型配置详情页面可以显示非敏感失败摘要，但离开页面或应用重启后即丢失；持久模型配置状态只保留是否就绪、最近测试成功时间、默认引用和必要的非敏感配置字段。这样减少可重建诊断状态的迁移和清理复杂度，同时让测试失败只作为当前修正动作的即时反馈。

本决策修订 0067、0138、0141、0142、0143、0144、0145、0151、0176、0177、0193、0197、0198 中关于“保存、覆盖或清除模型配置测试失败摘要”的持久化假设。相关错误类型、HTTP 状态码和简短说明的归一化规则仍用于当前页面展示；测试失败仍会按既有规则清除就绪状态、最近测试成功时间和默认引用。
