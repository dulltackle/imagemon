# 图片数量映射为一次原子模型调用

生成任务的图片规格数量 N 在首版映射为对模型服务的一次请求（OpenAI 兼容的 `n` 参数），而不是 N 次模型调用。因此一次图片任务始终对应一次模型调用，与 0029（全应用同一时间只允许一个模型调用）保持"一次任务=一次调用"的清晰对应，不在任务内部引入 N 次串行调用、部分中断或锁占用分叉等复杂语义。

由此任务状态没有"部分成功"，只在 0024 的进行中、完成、失败、状态未知之间取值：

- 模型调用整体失败 → 失败，按 0025 保存结构化错误摘要，不保存任何图片结果。
- 模型调用成功且返回 M≥1 张图片 → 完成，按 0180 保存实际返回的 M 张图片结果（即使 M 小于请求的 N，例如模型自身封顶）。任务快照仍记录请求的数量 N，用于解释当时的请求与实际产出的差异。
- 模型调用成功但返回 0 张可用图片 → 失败，按 0025 保存错误摘要，不保存图片结果。

后续若要支持"逐张产出 / 部分成功"的多图模式，需要重新评估任务状态枚举、错误摘要结构和全局锁语义，并另行记录决策。
