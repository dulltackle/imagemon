# 升级名称冲突时抑制新增内置条目，保留个人条目

应用升级后，当前版本的内置图鉴条目以应用包为准（0021）。如果某个新增内置图鉴条目的名称与设备上已有的同名个人图鉴条目冲突——这种冲突由升级引入，而不是使用者在写入时造成——应用保留该个人图鉴条目原名不动，并且本版本不把与之同名的内置条目加入合并图鉴，直到使用者主动改名该个人条目后，被抑制的内置条目才在合并图鉴中显现。应用可以提供一次性提示，告知存在因同名而被抑制的内置条目。

这样裁决是因为升级时其余各条原则无法同时自动满足：0020 要求个人条目不被隐藏或停用，0035 要求不自动改名、不静默覆盖使用者数据，而 0010 要求合并图鉴名称全局唯一。三者权衡下，优先保护使用者已有数据：不动个人条目、不自动改名，用"暂不加载冲突内置条目"维持合并图鉴唯一性。任务历史不受影响，因为任务快照按 0036/0098 保存执行时的条目身份，不依赖当前合并图鉴。

这是 0010 唯一一处由"使用者写入"以外的力量（应用升级）触发的名称冲突场景，故单独记录其裁决方向。后续若内置条目集合策略变化，需重新评估该抑制规则。

抑制是原子且仅运行时的：被抑制的内置条目整体不出现在合并图鉴中，其正文、展示信息（含 0050 的应用约定分类、搜索标签、任务类型标识）以及 0046 的内置封面示例都一并不显现，任何属性都不会回退或泄漏到同名个人条目上。内置封面作为应用包内只读资源仍随包分发（构建期无法预知某设备是否存在同名个人条目），抑制期间只是未被使用。使用者改名个人条目解除抑制后，该内置条目连同其约定分类和内置封面原样显现，不做回退、合并或部分加载。
