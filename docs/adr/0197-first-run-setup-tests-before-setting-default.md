# 首次设置内联显式测试，通过才设为默认

首次设置（0065）帮助使用者创建默认图片模型配置和默认文本模型配置时，必须内联一次显式测试连接，只有测试通过的配置才被设为对应类型的默认模型配置。测试失败、被跳过或被取消的配置照常保存为未就绪模型配置，但不被设为默认；对应类型的图片任务或模板提炼入口按 0065 显示需要配置。

本决策消除 0065 与默认就绪铁律的冲突。默认模型配置必须始终指向就绪模型配置（0131），就绪只能由字段完整、已保存 API Key 且最近一次显式测试通过得到（0128/0033）。0065 描述首次设置「创建默认图片模型配置和默认文本模型配置」，却未提测试连接；若首次设置直接把未经测试的配置设为默认，将违反 0131「未就绪不能默认」的不变量。唯一合法路径是首次设置内联一次显式测试，到达就绪后再设默认。

首次设置中的测试连接复用既有测试连接语义，不另立一套：受全局模型调用锁约束（0029）、使用 30 秒应用侧超时（0151）、按 0193 只验证连接可达与凭据被接受而不依赖 `/models` 列出模型名、按 0138 归一化失败摘要、可取消本地等待（0146）、中断后按 0150 恢复测试前状态。首次设置允许文本模型配置复用图片模型配置的连接信息（0065），但凭据按 0066 写入各自独立副本，两个配置若都要设为默认，各自都需要一次测试通过。

首次设置可以跳过（0065）；跳过、测试失败或取消都不会设默认，也不会因此阻断完成首次设置。使用者后续可在设置页补全或修正配置、显式测试通过后再设为默认（0127/0128）。首次设置不绕过 0136（凭据不可明文查看）、0130/0144（替换凭据需重测）等既有规则。

后续若要为首次设置提供「先保存全部配置、最后统一测试」等批量编排，需要重新评估全局锁下的串行测试顺序与失败处理，并另行记录决策。
