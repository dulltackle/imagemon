# 业务模型调用只锁定自身写入目标，不冻结其余本地写操作

业务模型调用（图片任务、模板提炼）进行中时，应用不冻结与本次调用无关的本地写操作，只保护本次调用最终写入的目标对象。这与 ZIP 导出（0106）、ZIP 恢复（0083）进行中的整库写冻结形成明确对照：导出和恢复原子地读取或写入整个本地存储，必须冻结一切并发写；而一次业务模型调用只触碰它自己的写入目标，依靠调用前快照纪律（0011/0023：图片任务在发起调用前已把图鉴条目内容、来源类型、任务输入、图片规格、模型配置快照和完整提示词全部落入不可变任务快照）与其余资产隔离，因此不需要、也不应当冻结整个应用。

**进行中业务调用的写入目标受保护，调用 resolve 前不可删除或修改：**

- 图片任务进行中时，本次的进行中任务历史及其任务快照不可删除或修改，直到调用完成、失败或（中断后）转为状态未知（0024）。这样模型调用返回时，结果（成功的图片结果弱引用、失败的错误摘要、或中断后的状态未知）始终落在一个仍然存在的任务历史上，不产生孤儿结果或写入丢失。
- 模板提炼重新生成方案的模型调用进行中时，承接结果的那一个提炼草稿（0031）不可丢弃或修改，直到调用结束或（中断后）按 0030 保留为结果不确定草稿。

**其余无关本地写操作照常允许，不被业务调用阻止：**编辑被使用图鉴条目的展示信息（0053，展示信息不进完整提示词，任务快照已捕获执行时值）、单条导入与单条导出（0005/0044）、修改应用默认规格（0038，本次任务已锁定执行规格）、删除其它图片结果或无关任务历史（0014）、以及本地确认写入提炼草稿成个人图鉴条目（0184，非模型调用）。这些操作因调用前快照纪律而数据安全，无需纳入冻结。

本决策只约束本地写操作边界，不改变并发模型调用规则：全局模型调用锁（0029）仍保证业务调用进行中不能并行发起其它模型调用，导出/恢复进行中（0083/0106）的整库写冻结规则不受影响，业务调用进行中与导出/恢复进行中也按各自规则互斥（业务调用占用全局锁时，导出/恢复确认按钮按 0083/0106 禁用；导出/恢复进行中时，业务调用按其禁止清单不能发起）。

后续若业务调用引入逐张产出、流式或部分结果（当前被 0028/0181 排除），写入目标的保护范围需要重新评估并另行记录决策。
