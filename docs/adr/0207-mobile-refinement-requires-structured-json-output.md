# 手机端模板提炼只接受结构化 JSON 输出

手机端模板提炼调用文本模型时，要求模型返回应用可解析的结构化 JSON，且只接受 JSON；应用不从自然语言、Markdown 或混合响应中猜测提炼方案。JSON 至少必须包含拟写入的 Promptdex 模板字段（名称、描述、可选版本、输入声明和逐字正文）、任务类型推断说明、保留规则摘要、删除规则摘要和提炼补充列表；解析成功后应用再执行 Promptdex 契约校验、保存提炼方案并展示逐字批准页面。

如果模型返回内容不是合法 JSON、结构不符合提炼方案契约，或无法通过 Promptdex 契约校验，本次生成方案视为失败并保留可重新生成的提炼草稿；草稿保存外部完整提示词、计划用途、草稿状态和非敏感文本模型配置快照，但不保存不可解析原始响应、散文内容或半截 JSON。这个约束刻意不同于 Builder skill 的 Agent 可读方案流程：手机端面对设备使用者自配文本模型，必须用机器可验证结构降低注入、遗漏和歧义落盘为个人图鉴条目的风险。
