# 模型名称使用自由输入并可选拉取列表辅助

手机应用首版中，图片模型名和文本模型名都是自由文本输入，不依赖兼容平台的模型列表接口。应用可以提供“拉取模型列表”作为辅助选择，但拉取失败不影响手动保存模型配置；拉取模型列表属于模型调用，受全应用单模型调用锁约束。拉取模型列表可以使用发起时模型配置详情页中的当前表单值，包括尚未保存的 base URL 和 API Key，但不会自动保存任何字段或 API Key。若配置已有保存的 API Key 且替换输入框为空，拉取模型列表使用已保存凭据；只有用户在替换输入框填入新 API Key 时，才使用该未保存新值发起本次拉取；如果既没有已保存凭据，也没有当前输入的新 API Key，则提示缺少凭据。拉取模型列表进行中时，模型配置详情页的配置名称、base URL、模型名、调用选项和 API Key 等字段保持只读，删除入口禁用，避免请求使用的表单值和页面继续编辑后的显示值分叉；首版不提供取消本地等待入口，用户需要等待拉取成功、失败或超时后再编辑或删除配置。拉取模型列表使用默认 30 秒应用侧超时，首版不提供用户可配置项；超时后释放全局模型调用锁，只在当前详情页即时提示，不保存模型配置测试失败摘要，不更新最近测试成功时间或最近测试结果，也不改变模型配置的就绪状态或默认身份。应用在拉取模型列表进行中被系统杀掉、崩溃或异常退出时，下次启动释放全局模型调用锁，丢弃未完成拉取，不保存失败或未完成记录，不改变模型配置状态，也不显示专门的一次性提示；用户需要时重新进入详情页显式拉取。拉取成功后的模型列表只用于当前选择会话，离开模型配置详情页或重新进入后不长期保留，用户需要再次显式拉取。拉取失败只在当前详情页即时提示，不保存模型配置测试失败摘要，不更新最近测试成功时间或最近测试结果，不改变模型配置的就绪状态或默认身份；用户仍可手动输入模型名并保存，模型调用锁在拉取结束后释放。已知模型执行本地能力校验，未知模型在执行时透传给兼容平台。
