# 备份数据表镜像合并图鉴并附展示图

表格备份最初只镜像个人图鉴条目（ADR 0213 时代的边界），备份数据表因此只承担「个人数据备份」职责。为了让每个图鉴条目在表里附一张对应图片，我们决定把镜像对象扩为**合并图鉴全量**：全部内置条目与个人条目都入表，记录携带「来源类型」（`built-in`/`personal`，与代码 `PromptdexCatalogEntrySourceType` 字面量一致）；满足「已生成图鉴条目」定义的记录（不分来源）附其最新一张图片结果作为展示图，未生成条目展示图留空。曾考虑只让已生成的内置条目入表（个人条目管备份、内置条目只为挂图），但选择了全量入表，让备份数据表同时成为完整的图鉴目录。

展示图是**单向投放的展示物**：原图直传不压缩，靠契约文本字段「展示图标识」（最新图片结果 id）进入既有镜像文本 diff——标识变更才触发上传或清空附件，附件内容本身不参与比对，使用者在飞书侧的改动不被察觉也不被修复，展示图永不回流本机。镜像保持无状态，失败语义维持「任一步失败即中止」，靠标识 diff 幂等补传；实现上必须先传图成功、后写标识，否则会出现永久性缺图。

后果：内置条目随应用版本增删会引起备份数据表记录变动；恢复必须按来源类型过滤，`builtin` 记录不参与写入（见词条「表格恢复」）；本决定的前提是个人授权码通道可上传附件，实现前需 spike 验证，失败则回退重议本决定。
