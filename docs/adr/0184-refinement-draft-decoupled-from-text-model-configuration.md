# 提炼草稿与文本模型配置解耦

提炼草稿（0030/0031/0074）按其状态区分两类操作对就绪文本模型配置的依赖：

- **写入个人图鉴条目是本地操作**。当草稿已含提炼方案时，使用者审阅、批准提炼补充并把方案确认写入个人图鉴条目，不发起模型调用，因此不要求当前选中或存在任何就绪文本模型配置。这与 0030（提炼不是图片任务）、0031（确认写入后清除草稿）一致。
- **重新生成方案是模型调用**。只有重新生成提炼方案才是模型调用，需要使用者在模板提炼页选中一个就绪文本模型配置，并受 0132–0134（保留并禁用 / 清空已删除 / 保留改名）约束。

提炼草稿按 0074 只保存文本模型配置快照中的配置名称和模型名，作为"当时用的是哪个"的说明性上下文，不是对配置实体的活引用。恢复草稿后重新生成时，应用不按该名称自动恢复或预选模型配置，而由使用者重新选择；因为配置名称在类型内唯一（0073）但同名不代表同一实体，按名静默绑定可能落到一个已被改动或重建的同名配置上，违背 0132 防止"以为在用原配置、实际悄悄切换"的原则。
