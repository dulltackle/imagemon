# 编辑任务输入文件在调用前本地校验

编辑任务的 `image` 和 `mask` 文件输入在使用者点击执行之前先做基础本地校验：文件可读取、确为图片、不超过应用固定的安全上限（具体的可接受范围与裁决方式见 0183）。校验不通过时，应用要求使用者重新选取或修正输入，不创建任务历史、不发起模型调用、不产生错误摘要。

这与 0039（不兼容图片规格要求使用者在调用前修正、不入历史）采用同一思路，把"可在本地预先发现的输入问题"与 0023（模型调用前先建立任务历史）区分开：0023 针对的是已经决定发起的模型调用，而本地可预知的输入问题不应污染任务历史，也避免为可预测的输入错误反复产生 0025 的失败摘要。服务端在调用后才暴露的拒绝仍按失败任务处理并保存错误摘要。
