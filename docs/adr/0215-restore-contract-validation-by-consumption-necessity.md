# 恢复契约校验按消费必需性分级

备份数据表的恢复方向校验原本要求契约完整（缺字段即失败、不补建）。契约扩展（新增「来源类型」「展示图标识」文本字段与「展示图」附件字段）后，若新字段一律必需，会锁死换机场景：旧契约表（v0.11.0 建）在新手机上既无法恢复（缺字段失败），又绝不能"先备份升级表结构"——空图鉴按镜像语义上传会删光表中所有记录，备份就此蒸发。

因此恢复校验从「契约完整性」转向「消费必需性」：只有恢复真正消费的字段才是恢复方向的必需字段。原有 7 个文本字段照旧从严；缺「来源类型」时整表按全 `personal` 处理（旧契约表只可能由旧版应用写入个人条目，该推断严格成立）；缺「展示图标识」「展示图」无影响（恢复不读它们）。备份方向不变，缺失字段自动补建，下次备份即把旧表升级到新契约。曾考虑增设契约版本号字段显式区分新旧表，但版本号同样可能被使用者改坏、还需维护版本校验矩阵，收益配不上复杂度。

代价：新表被使用者删掉「来源类型」后不会被当场拒绝，内置记录会以个人候选身份进入恢复预检，但会撞上既有「与内置条目重名」冲突通道被明示排除，不会静默写坏；下次备份补建字段自愈。将来契约再扩展时，恢复方向默认按「恢复是否消费该字段」判定必需性，而非自动纳入必需集。
