# 明确离线时在调用前拦截模型调用

设备明确离线（系统报告无网络连接）时发起图片任务或模板提炼，应用在调用前提示"无网络连接"并阻止本次发起，不创建任务历史、不发起模型调用。这与 0039（不兼容规格）、0179（输入文件）、0183（格式/大小）一致：网络可用性是应用能可靠本地获知的前置条件，应在调用前纠正，而不是为可预测的离线失败在任务历史里留下记录。

系统报告在线但服务端实际不可达（DNS、连接被拒、超时等）只能通过尝试调用才能发现，这类情况仍按失败任务处理并保存 0025 错误摘要，不属于本决策的调用前拦截范围。

诊断类模型调用——模型配置测试连接和拉取模型列表——本就不进入任务历史；明确离线时它们在对应模型配置详情页即时提示无网络连接，与其它即时失败一致，不生成待查看提示，也不改变模型配置就绪状态或默认身份。
