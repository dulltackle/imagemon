# 编辑输入本地校验只做最小检查与固定安全上限

0179 的调用前本地校验只做最小检查：文件可读取、确为图片，且不超过应用固定的安全上限（用于防止内存溢出或崩溃的文件大小与像素上限）。具体支持哪些图片格式、多大尺寸真正可用，不在本地裁决，而是交由模型服务；服务在调用后拒绝时按 0179 作为失败任务处理并保存 0025 错误摘要。

这样选择是因为模型配置允许自定义任意 OpenAI 兼容服务（0032），应用无法预知各服务真实的格式与大小限制。若本地强制固定的格式白名单或偏紧的大小上限，会错误拒绝某些服务本可接受的输入；只保留宽松的安全上限可在防崩溃与不误拒之间取得平衡。

安全上限由应用固定，首版不提供用户可配置项，也不进入备份或恢复，沿用 0152 减少设置复杂度、避免额外默认值与恢复边界的思路。后续若需按服务区分限制或开放配置，需另行记录决策。
