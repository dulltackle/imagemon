# Promptdex Markdown 仅作为交换格式，不再承担任何存储或备份载荷

ADR 0206 已确立个人图鉴条目本地持久化以结构化字段为真相源。本决策把该原则推广到全部数据通道：Promptdex Markdown 从此只在导入、导出和复制边界上作为交换格式存在（图鉴条目导出、图鉴条目 Markdown 复制、内置图鉴条目的随包分发、模板提炼的产出解析），任何存储形态和备份载荷都不得以 Markdown 文本为准。

直接后果是表格备份与表格恢复只读写结构化字段：备份数据表中不设 Promptdex Markdown 列，恢复校验直接在结构化字段上执行既有契约校验，从而避免同一条记录里出现「结构化列与 Markdown 列打架、恢复时不知以谁为准」的双来源问题。曾考虑的替代方案是以无损 Markdown 列作为唯一恢复载荷、结构化列仅作展示，但在真相源已经是结构化字段的前提下，这会让备份链路多一次序列化/反序列化往返，并把使用者在表格里的合法编辑（展示信息整理）排除在恢复之外，故放弃。

若未来需要新的持久化或备份通道（如其他云表格、文件同步），同样必须以结构化字段为载荷；若要推翻本决策改回 Markdown 载荷，需另立 ADR 说明理由。
