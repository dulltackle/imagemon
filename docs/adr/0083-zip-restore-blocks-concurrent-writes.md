# ZIP 恢复期间禁止并发写操作

ZIP 恢复在使用者确认预检结果后进入恢复进行中状态。已有模型调用进行中时，应用可以允许使用者进入恢复预检页查看将导入的内容和冲突处理项，但必须禁用确认恢复，直到当前模型调用结束。恢复预检页停留期间必须实时跟随全局模型调用锁状态；只要图片任务、模板提炼、模型配置测试连接或拉取模型列表进行中，确认恢复按钮就保持禁用并显示当前模型调用进行中的原因；模型调用结束后，应用必须基于当前本地状态重新校验预检结果，如果预检内容仍有效，再自动恢复可确认状态。使用者确认恢复前，如果冲突项、改名结果、可导入判断或需要确认的事项发生变化，应用必须回到更新后的预检结果并要求使用者重新确认。恢复进行中期间，应用必须阻止图片任务、模板提炼、模型配置测试连接、拉取模型列表、图鉴编辑、单条导入导出、图片结果删除、任务历史删除、应用默认规格修改等会修改应用数据或触发模型调用的操作。恢复进行中可以展示恢复进度和失败结果，但不能与其他写操作交错执行，以避免恢复事务、文件暂存提交和使用者操作产生不一致状态。
