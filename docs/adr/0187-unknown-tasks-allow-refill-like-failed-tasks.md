# 状态未知任务与失败任务一样可重新填写

状态未知任务（0024）与失败任务（0026）采用同一套重新填写规则：仅当当前图鉴条目仍存在，且历史任务快照可被当前版本解析出可匹配输入时，提供重新填写入口，打开新的任务表单并把历史输入作为可编辑预填建议；使用者确认后创建全新的任务历史，原状态未知记录保持不变。当前图鉴条目不存在，或恢复导入的旧任务快照无法被当前版本解析为可匹配输入时，不能从该状态未知任务重新填写。

重新填写不附带特殊警示。状态未知意味着模型服务可能已处理并收费但应用没有可靠结果可保存（0024），然而失败任务同样可能已产生费用，二者在这一点上没有区别；重新填写是由使用者主动发起、创建全新任务的动作，不是 0024 禁止的自动重试，因此沿用 0076/0122 少警示的风格，不额外提示重复或计费风险。

应用不尝试恢复服务端可能已生成、但应用从未收到的图片（首版无流式或部分预览，0028），也不按 0097 手动更正状态；状态未知是终态，唯一向前的路径就是这条创建全新任务的重新填写。
