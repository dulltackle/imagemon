# 模型配置测试连接是显式模型调用

手机应用首版在模型配置详情页提供显式“测试连接”动作，但保存模型配置时不自动测试，测试失败后也不自动重试。失败摘要详情页允许用户再次手动触发测试连接，用于在修正配置或等待服务恢复后重新验证；用户未修改配置时也允许重试。每次重试仍是显式模型调用，受全应用单模型调用锁约束，并更新测试状态：失败会覆盖最近失败摘要，成功会清除旧失败摘要并保存最近测试成功时间；无法确认兼容性也会清除旧失败摘要，但只在测试完成后的当前详情页即时显示普通未就绪说明，不作为最近测试结果持久保存。离开该详情页再返回时不保留这条即时说明，只显示普通未就绪状态。已就绪配置再次测试得到无法确认兼容性时，清除就绪状态和最近测试成功时间；如果该配置是默认模型配置，也清除默认身份。测试连接进行中时，失败摘要详情页的重试入口禁用，不能再次发起并行测试；用户仍可取消当前本地等待。测试结果只用于提示配置可用性，不决定配置是否可以保存，也不写入任务历史；其他模型调用进行中时不能并行发起。

测试连接不调用图片生成接口，也不生成模板提炼方案；优先调用 OpenAI 兼容的 `/models` 或等价轻量探测接口。首版测试连接只验证 base URL、API Key 和模型名存在，不额外检查模型是否声明支持图片生成或图片编辑能力。如果轻量探测成功且返回可解析模型列表，但列表不包含当前配置的模型名，测试连接按失败处理，不能标记配置就绪。模型名按服务端返回的原文做精确匹配，大小写不同也视为不匹配。如果兼容平台不支持轻量探测，或返回内容无法解析为可确认的模型列表，应用报告无法确认兼容性但不阻止保存；该结果不视为测试通过，不保存最近测试成功时间、测试时间、最近测试结果或新的模型配置测试失败摘要，并清除已有模型配置测试失败摘要，不标记配置就绪，配置不能设为默认，也不能用于图片任务或模板提炼；离开详情页再返回或应用重启后不保留专门的无法确认兼容性说明，只显示普通未就绪状态。真实执行时仍按图片任务或模板提炼流程处理错误并保存相应摘要。
