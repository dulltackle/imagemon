# 一条任务历史弱引用多张图片结果

当图片规格的数量大于 1 时，一次生成任务产出多张图片结果。这些图片结果都是独立、可单独管理的资产，并都按 0015 弱引用同一条任务历史。任务历史详情把它们作为一组结果展示；其中任一张被删除时该位置显示图片已不存在，不影响其余结果，也不影响任务历史本身（沿用 0014 的独立删除与 0015 的弱引用）。

图片任务成功后的待查看提示（0169）以"这一次生成"为单位：打开该次任务历史详情，或打开这组结果中任意一张图片结果详情，都视为该次生成已被发现，立即清除图片入口和历史入口两边的标记，而不是为每张结果维护独立未读项。这与 0166（待查看提示不显示计数）一致。

此决策把 0015 和 0169 中隐含的"一次任务对应一个结果"显式确定为一对多，避免后续在删除级联、缺失资产展示和待查看标记清除上出现一对一假设。
